# Phase 3 Plan: Agentic Metacognition & Swarm Reliability

This document outlines the strategic shift from **Phase 2 (Isolated Model Benchmarking)** to **Phase 3 (Agentic Swarm Reliability)**.

## The Problem: The "Isolated Benchmark" Gap
Our Phase 2 work (Kaggle Submission) successfully isolated the **M-Ratio** of individual models. However, in our local "Resilient Swarm" setup:
1.  The Mediator, Hand, and Critic are working together to solve the `chandra_packet`.
2.  The benchmark server (A2A) is currently stateless and isolated — it measures the models *as if they weren't in a swarm*.
3.  **The Gap**: We are not yet measuring how the **process** (the collaboration) handles metacognitive failures.

## Phase 3 Objectives

### 1. Unified Bayesian Swarm Probes
Instead of isolated 2-turn trials, move to **N-Turn Bayesian Tracking**:
*   Track the confidence of the **Mediator** as it receives feedback from the **Critic**.
*   Calculate the "Swarm M-Ratio": How well does the collective output correlate with the actual correctness of the code generated by the Hand?

### 2. Adversarial Process Injection
Inject "Gaslighting" not just into the model prompt, but into the **Swarm Protocol**:
*   The **Critic** is forced to give "Negative Evidence" (false criticism) to a correct **Hand** output.
*   We measure the **Mediator's** ability to resist this "Social Pressure" and uphold the correct prior.

### 3. Integrated A2A-v2 Server
Upgrade the local A2A server from stateless probes to the **Metacog-Multiturn-V2** logic, making it a "live" participant in the research loop.
